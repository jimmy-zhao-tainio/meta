# Meta Consistency Audit - 2026-06-16

## Summary

This audit treats `meta` as the source of truth and checks the current working tree after the stale `ExportSqlAsync` removal. No production cleanup was performed as part of this audit report.

The broad `meta` service, CLI, docs, and test surfaces are mostly aligned after removing public APIs that had no clear functional caller. The `meta-bi` sweep found substantial dependency on `Meta.Core` and `Meta.Adapters`, especially generated tooling contracts, but no use of the removed SQL export adapter surface.

## Findings

1. Stale adapter/service methods were removed from current production surface.
   - Removed: `ExportSqlAsync`, `ExportCSharpAsync`, `ValidateIncremental`, `ApplyWithoutHistory`, `GetUndoOperations`, and production `GenerationService.AreEquivalent`.
   - `BuildManifest` remains only as a private implementation detail.

2. `ExportCSharpAsync` was removed for the same reason as the SQL export adapter method.
   - The CLI uses `GenerationService.GenerateCSharp`, not `IExportService.ExportCSharpAsync`.
   - Its only scoped consumer was an adapter-only test.

3. `ValidateIncremental`, `ApplyWithoutHistory`, and `GetUndoOperations` were removed because no current functional caller justified keeping them.

4. `BuildManifest` and `AreEquivalent` were tightened.
   - `BuildManifest` remains only as a private implementation detail used by generation methods.
   - `AreEquivalent` was removed from production; tests now compare manifests locally.

5. `meta-bi` depends heavily on generated tooling and serializer contracts.
   - Filtered project scan: 84 project files, 50 `Meta.Core` references, 18 `Meta.Adapters` references.
   - Generated `*.Tooling` projects frequently reference `Meta.Adapters` and `Meta.Core`.
   - Many `meta-bi` generated model classes call `TypedWorkspaceXmlSerializer`, `LoadFromXmlWorkspace`, and `SaveToXmlWorkspace`.
   - Recommendation: do not change generated tooling shape or serializer method names without a coordinated `meta-bi` regeneration/build pass.

6. `meta-bi` has no observed dependency on the removed stale Meta foundation surface.
   - Filtered stale-symbol search for `ExportCSharpAsync`, `ValidateIncremental`, `ApplyWithoutHistory`, `GetUndoOperations`, and `GenerationService.AreEquivalent` returned no relevant source matches.
   - The only `AreEquivalent` hits are unrelated `SqlServerDefaultExpressionComparison.AreEquivalent` calls inside `MetaSql`.
